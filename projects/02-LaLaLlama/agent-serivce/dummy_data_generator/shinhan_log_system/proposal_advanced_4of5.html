제안서: 신한투자증권 로그정보시스템 고도화 – 혁신-안정성 균형형 제안(가상)

제안사: NexWave Solutions, 작성일/담당자: 2025-10-01 / 이수현 이사


1. 제안 개요

  • 배경 및 필요성
  • 메타 차세대 거래 로그와 표준 스키마 연계 필요성 증가에 따라, 사용자 친화적 UX 기반의 실시간 조회 및 강력한 컴플라이언스 제어가 요구됩니다.
  • 다중 그룹 권한(IT/일반직원) 및 민감정보 관리 강화가 필수인 상황에서, 승인-요청-만료 라이프사이클 관리와 감사 추적을 내재화한 시스템이 필요합니다.
  • 목표
  • 사용자 중심 UX를 통해 로그 탐색, 검색, 필터링 경험 개선
  • 컴플라이언스 이슈 대응 강화: 민감정보 항목 승인/권한 통제 및 감사 추적 강화
  • 그룹 권한에 따른 로그 조회 제한 및 차등 정보 노출 구현
  • 차별화 포인트
  • 메타 로그의 표준 스키마/메타데이터 정합성 보장으로 거버넌스 강화
  • 데이터 마스킹, 필드 단위 접근 제어, 감사 로그의 강력한 포렌식 추적
  • AI 기반 태깅/카테고라이제이션으로 검색 시나리오 자동 보강
  • 단계별 KPI(범위 지향, 측정 방법 포함)
  • 구축 직후: 가용 97.0%, 평균 응답 1.8s(단순 조회 기준), 동시접속 2,500명, TPS 50,000~60,000건/시
  • 3개월: 가용 97.5%, 평균 응답 1.7s, 동시접속 2,800명, TPS 60,000~70,000건/시
  • 6개월: 가용 98.0%, 평균 응답 1.6–1.8s(복합 조회 포함), 동시접속 3,000명, TPS 80,000~100,000건/시
  • 데이터 처리: 배치 30분~1시간, 실시간 이벤트 지연 2–5초, 데이터 정합성 99.0–99.5%
  • 안정화/운영: 2개월간 전담 안정화 팀 상주
  • 전제 조건
  • 정상 운영 시, 표준 유지보수 제외
  • 메타 신규 거래로그의 표준 스키마 및 연계 인터페이스 확정
  • SSO/IAM 및 보안 정책의 현행 규정 준수 유지

  • Case Gamma (보험사 빅데이터 로그 대시보드)
  • 규모/성과: 배치 처리 1시간 이내, 데이터 정합성 99.2%, 2,500명 동시 처리

  • 주요 성과: 메타데이터 정합성 확보, 공통코드 유효값 연동, 로그 상세 화면 설계 지원
  • 시사점
  • 금융권에서도 메타 로그 표준화, 접근 제어 강화, 감사 추적 강화가 핵심 차별점으로 작용함
  • 클라우드 네이티브 아키텍처와 자동화된 QA/보안 테스트 도입이 높은 안정성과 품질을 뒷받침함

3. 프로젝트 수행

3.1 기술 부문

  • 시스템 아키텍처 개요
  • 데이터 흐름: 거래 로그 수집(메타로그) → 메시징 큐(Kafka) → 스트리밍/배치 처리(Flink/Spark) → 데이터 저장/인덱싱(데이터 레이크 + Elasticsearch) → API 계층(Spring Boot) → 프론트엔드(React)
  • UI/UX: 사용자 친화적 대시보드, 필터링/저장된 쿼리 재사용, 다중 그룹 권한에 따른 화면 제어
  • 인증/권한: OAuth2/OIDC SSO 연계, 세분화된 권한 정책, 요청-검토-승인-만료 라이프사이클 관리
  • 보안/컴플라이언스: 민감정보 마스킹(AES-256), 접근로그 3년 보관, 감사 추적 테이블 강화, 데이터 암호화 at rest/in transit
  • 기술 스택 및 선정 근거
  • Spring Boot: 엔터프라이즈 필요 기능, 보안 패치 신속성, 대규모 조직 통합 용이
  • React: 컴포넌트 재사용성, 금융권 대규모 UI 사례 다수
  • Kafka: 신뢰성 높은 이벤트 스트리밍, 로그 이벤트 처리에 최적
  • Elasticsearch: 로그 검색 및 필드 기반 검색 최적화
  • Redis: 실시간 캐시/세션 관리
  • Kubernetes: 클라우드 네이티브 운영, 자동 확장/롤링 업데이트
  • 성능 목표 및 측정 방법
  • 가용률: 97.5% ~ 98.0% (1차년도 97.0% → 2차년도 98.0% 목표), 측정: Grafana + Prometheus + Uptime Robot 연동
  • 평균 응답시간: 1.5–2초(단순 조회 1.5초, 복합 조회 2초), 측정: APM 도구(Pinpoint/Elastic APM)
  • 최대 응답시간: 5–6초(복잡 쿼리 기준)
  • 동시 접속자: 2,500명 이상, 측정: 부하테스트(JMeter)
  • 시간당 처리량: 60k–100k 건/시(로그 이벤트/검색 쿼리 포함), 측정: JMeter/ Gatling
  • 데이터베이스 쿼리 응답시간: 일반 쿼리 100–200ms, 복잡 쿼리 200–300ms
  • 배치 처리 시간: 30분~1시간(일별 집계 기준, 병렬 처리 적용)
  • 실시간 이벤트 처리 지연: 2–5초
  • 데이터 정합성: 99.0–99.5% (자동 검증)
  • 측정 도구 및 방법
  • 가용/응답 모니터링: Grafana/Prometheus + Uptime Robot
  • 트랜잭션/분석: Pinpoint/Elastic APM
  • 부하테스트: Apache JMeter
  • 데이터 일관성 검증: 커버리지 기반 자동 검증 스크립트

3.2 테스트 전략
