solution: |
  1. 다운된 WAS 서버 2대 긴급 재기동
  2. 나머지 2대 서버로 트래픽 분산
  3. 메모리 덤프 분석 (jmap, MAT 도구 사용)
  4. 메모리 릭 원인: 세션 객체 미해제
  5. 임시 조치: Heap 메모리 4GB → 6GB 증설
  6. 근본 해결: 세션 타임아웃 설정 조정 (30분 → 10분)
impact: 인터넷뱅킹 접속 지연 1시간, 약 5,000명 이용 불편
resolution_time: 1시간
contact_person: 김영수
contact_email: kim.ys@suhyup-bank.com
lessons_learned: |
  - WAS 메모리 사용률 80% 이상 시 사전 알림
  - 주기적 메모리 프로파일링 필요
  - 세션 관리 정책 재검토
  - WAS 클러스터 최소 4대 이상 권장
