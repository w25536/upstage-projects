1. 제안 개요

1-1. 추진 배경
- 수협은행은 온·오프라인 채널의 유기적 연계와 실시간 개인화 마케팅을 통한 고객 관리 효율성 제고를 목표로, 안정성과 신뢰성을 최우선으로 하는 데이터 기반 옴니채널 마케팅 인프라 도입이 필요합니다.
- 본 제안은 이미 검증된 기술 스택을 활용하여 리스크를 최소화하고, 대면·비대면 채널의 실시간 데이터 흐름, 캠페인 설계/실행/분석의 전체 사이클을 안정적으로 운영하는 것을 목표로 합니다.
- 추진 방식은 1단계 인프라 안정화, 2단계 캠페인 엔진 운영, 3단계 실시간 맞춤형 마케팅까지의 단계적 도입으로, 초기 2~3개월 내 안정화 계획과 함께, 2~3개월 간의 시스템 안정화 기간을 거쳐 운영을 고도화합니다.

1-2. 목표 및 핵심 KPI
- 서비스 가용률: 97.0% ~ 97.5% 목표 (운영 안정화 기간 포함)
- 응답 시간: 평균 3.0초, 최대 8초 내외
- 처리량: 시간당 8만 건 목표
- 데이터 처리 지연: 실시간 8~12초, 배치 1.5시간 내외
- 캠페인 발송률: 88%
- 오픈율: 22%
- 전환율: 5%
- 시스템 안정화 기간: 3개월
- 장애 복구 시간: 45분 이내
- 백업 주기: 일 1회 전체 백업 + 주 3회 증분 백업
- 동시 접속자 처리: 1,200명 규모 안정 운영
- 데이터베이스 응답시간: 180ms 내외

1-3. 범위
- 실시간 수집 서버 구축: 인터넷뱅킹, 모바일뱅킹, 모바일웹, 거래 데이터, 상담 데이터의 실시간 수집, 정제, 가공 및 모니터링
- 캠페인 시스템: 배치형/실시간 캠페인 설계, 실행, 분석, 관리 모듈 구축
- 캠페인 DB 구축: 캠페인 마트(DB), 마케팅 운영 DB 설계 및 구축
- 데이터 및 시스템 연계: 발송 채널(SMS/이메일/푸시 등) 및 관련 시스템과의 연계
- 데이터 거버넌스 및 보안 체계 확립
- 초기 운영 기간 내 온전한 운영 매뉴얼, 교육 및 이관

2. 제안 업체 일반 현황

2-1. 제안사 개요
- 회사명: 한빛정보시스템(주)
- 설립 연도: 2008년
- 주요 사업: 금융권 마케팅 시스템 및 데이터 플랫폼 구축, 운영, 유지보수
- 직원 규모: 약 320명
- 연간 매출: 약 520억원(최근 3개년 연평균 성장률 6~9%)
- 인증/수상: ISMS-P, ISO 27001, 정보보호 관리체계 등급 인증 보유

2-2. 유사 프로젝트 수행 경험
- 사례 A: 국내 중대형 은행의 실시간 마케팅 플랫폼 구축(2019-2022)
  - 목적: 온·오프라인 채널의 데이터 통합, 실시간 의사결정 엔진 도입
  - 성과: 캠페인 발송률 85%, 오픈율 21%, 전환율 4.8% 달성
- 사례 B: 신용협동조합 계열 은행의 데이터 파이프라인 구축 및 캠페인 마트 도입(2021-2023)
  - 목적: 대용량 데이터 수집/정제/저장 및 캠페인 분석 환경 구성
  - 성과: 배치 지연 60분→80초 수준의 대폭 개선, 데이터 품질 이슈 40% 감소
- 사례 C: 대형은행의 옴니채널 캠페인 엔진 도입 및 운영 관리(2020-2024)
  - 목적: 배치 + 실시간 캠페인 통합 엔진, 다채널 발송 및 반응 분석
  - 성과: 채널 반응 분석 정확도 92% 이상, 시스템 가용률 97.2% 유지

2-3. 재무 현황
- 최근 3년간 매출: 2022년 420억원, 2023년 480억원, 2024년 520억원
- 영업이익: 2022년 58억원, 2023년 72억원, 2024년 86억원
- 현금성자산/단기차입금 비율 안정화 및 외부감사 보고서에 의거한 재무 건전성 양호
- 프로젝트 수행에 필요한 자금 여력 및 신용도는 은행권 표준 수준으로 확인 가능

3. 프로젝트 수행

3-1. 기술 부문

- 시스템 구축 전략
  - 검증된 아키텍처: 온프렘(코어 DB) + 프라이빗 클라우드/하이브리드로 구성, 이벤트 기반 아키텍처(MSA 미니 서비스) 채택
  - 실시간 데이터 파이프라인 중심: 데이터 수집-정제-저장-분석-캠페인 실행의 엔드투엔드 흐름을 Kafka를 중심으로 구성
  - 데이터 거버넌스 강화: 데이터 품질 체크, 메타데이터 관리, 데이터 카탈로그를 통한 사업부간 데이터 이해도 향상
  - 표준화된 운영 프로세스: 개발-운영 간 공통 산출물 및 산출물 서식, 변경 관리 체계 적용

- 서비스 안정화 방안
  - SRE 원칙 적용: SLI/SLA/SLO 정의 및 주기적 모니터링
  - 가용성 확보: 이중화, 장애대응 Runbook, Blue/Green 배포 및 자동 롤백
  - 장애 대응 시나리오: 장애 분류별 RTO/RPO 정의, 24x7 운영 지원 체계 확보
  - 모니터링/로깅: Prometheus+Grafana, ELK 스택으로 시스템 상태와 캠페인 흐름 실시간 감시
  - 재난 복구 계획: 지리적으로 분산된 백업 및 DR 테스트 연 1회 실시

- 시스템 구성
  - 핵심 구성:
    - 데이터 수집 서버: 실시간 스트리밍 수집, 데이터 정제 및 표준화
    - 캠페인 엔진: 배치형 캠페인 엔진, 실시간 캠페인 의사결정/발송 엔진
    - 캠페인 마트 DB: 대상 고객, 행동 데이터, 반응 데이터, 성과 데이터 저장
    - 마케팅 운영 DB: 캠페인 설계 정보, 대상 고개 정보 이력 관리
    - 데이터 저장소: 데이터레이크(HDFS) 및 데이터웨어하우스(전사 데이터마트), RDBMS(Oracle/PostgreSQL)
    - 채널 전달계: 이메일/SMS/푸시 채널 연계 모듈
    - 보안/거버넌스: IAM/암호화, DLP, 접근 제어, 데이터 마스킹
  - 기술 스택 예시:
    - 데이터 인제스션/스트리밍: Apache Kafka, Kafka Connect
    - 실시간 처리: Apache Flink 또는 Spark Structured Streaming
    - 배치/워크플로우: Apache Airflow
    - 데이터 저장: Oracle DB(코어), PostgreSQL(마트), HDFS/S3(데이터레이크), 데이터웨어하우스(예: PostgreSQL 기반 혹은 Amazon Redshift형 구조)
    - 캠페인 엔진: Drools 기반 비즈니스 규칙 엔진 + 커스텀 의사결정 엔진
    - 채널 연계: SMTP/Email Gateway, SMPP Gateway, FCM/APNs 등
    - 모니터링/로그: Prometheus, Grafana, ELK Stack
    - 보안/거버넌스: OAuth2/OIDC, SAML, KMS/HSM, 암호화, 감사로그
    - 데이터 카탈로그/거버넌스: Apache Atlas 또는 Amundsen 계열 도구
  - 인터페이스/표준화
    - 은행 내부 데이터 모델과의 매핑 표준 정의
    - 데이터 품질 규칙, 메타데이터 표준, API 계약(OpenAPI)

- 시스템 구축 방안
  - 단계적 구축: 4단계로 구분
    1) 인프라 및 데이터 수집 파이프라인 구축(1~3개월)
    2) 배치 캠페인 엔진 설계 및 기본 운영(3~5개월)
    3) 실시간 캠페인 엔진 도입 및 채널 연계(4~6개월)
    4) 데이터 마트/대시보드 및 UX 강화, 안정화(3~4개월)
  - 공통 산출물: 시스템 구성도, 데이터 흐름도, 요구사항 명세서, 상세 설계서, 테스트 계획/실행 체크리스트

- 성능 관리 방안
  - 용량 계획: 동시 접속 1,200명 규모 대비 예비 40% 여유 확보
  - 성능 목표: 평균 응답 3.0초, 99%ile 6초 이내
  - 모니터링: 24x7 알람 체계, 주기적 용량 재계획(분기 1회)
  - 캐싱 계층: Redis/LRU 캐시를 통해 자주 accessed 데이터 응답시간 최소화
  - 데이터 처리 지연 관리: 실시간 파이프라인 8~12초 지연 목표, 배치 파이프라인 1.5시간 내 지연 관리

- 테스트 방안
  - 테스트 범위: 단위/통합/성능/보안/회복력(test chaos) 포함
  - 자동화 커버리지 목표: 최소 70% 코드 커버리지, Cypress/Selenium 기반 UI 자동화
  - 성능 테스트: JMeter 또는 Gatling로 캠페인 발송 시나리오별 부하 테스트
  - 데이터 품질 테스트: 데이터 유효성 규칙 검사, 데이터 누락/중복 체크
  - 보안 테스트: 정적/동적 분석, 권한/인가 테스트, 세션 관리 점검

- 보안 관리 방안
  - 인증/권한 관리: OAuth2/OIDC, SAML, RBAC/ABAC
  - 데이터 보안: TLS 1.2+, 데이터 암호화 at rest/in transit, 필드 수준 마스킹
  - 비상대응: 백업/복구 절차, 재해복구 계획, 침해 사고 대응 프로세스
  - 규정 준수: ISMS-P/ISO 27001 등 국내 표준 준수, 감사 로그 보존

- 데이터 관리 방안
  - 거버넌스 프레임: 데이터 품질 관리, 데이터 계보/라인리지 관리
  - 메타데이터 관리: 데이터 카탈로그를 통한 데이터 이해도 향상
  - 데이터 품질: 자동화된 품질 점검 및 품질 이슈 트래킹
  - 데이터 수명주기 관리: 수명주기 정책, 보안 및 프라이버시 요구사항 반영
